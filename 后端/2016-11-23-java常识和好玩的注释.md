类与接口
基本原则：
一个类只做一件事情。另一个原则时根据每个类的职责进行划分，比如用
User来存放用户信息，而用UserDAO来对用户信息进行数据访问操作，用UserServer对用户
信息的业务操作等等。多个类中使用相同方法时将其方法提到一个接口中或使用抽象类，
尽量提高重用度。不希望被实例化的类的缺省构造方法声明为private。
乘法和除法尽量使用移位 << 和 >>
